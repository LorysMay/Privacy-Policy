Résumé

IPcamVision Pro est une application locale macOS permettant de visualiser, d'enregistrer et de gérer les caméras IP configurées par l'utilisateur, les NVR et les DVR. L'application ne collecte pas de données personnelles pour le développeur, n'utilise pas de services d'analyse, de suivi publicitaire ou de profilage, et n'envoie les données de votre caméra à aucun serveur nous appartenant. La configuration de l'appareil, les informations d'identification, les enregistrements, les instantanés et les préférences restent sur votre Mac, sauf si vous choisissez de les exporter ou de les partager.

Données gérées localement par l'application

L'application gère les données suivantes sur votre appareil car elles sont nécessaires pour fournir ses fonctionnalités :

  • Adresses et configuration des caméras, NVR et DVR, y compris les adresses IP ou les noms d'hôte, les URL RTSP ou de flux, les ports, les profils de flux, la marque PTZ, la disposition de la grille, les paramètres de lecture et les paramètres de détection de mouvement. Ces valeurs sont stockées dans les préférences privées de l'application à l'aide de UserDefaults et sont isolées par le bac à sable de l'application macOS.
  • Noms d’utilisateur ONVIF et noms d’utilisateur d’appareil. Ceux-ci sont stockés avec la configuration de l'appareil dans les préférences privées de l'application.
  • Mots de passe ONVIF et mots de passe des appareils. Les mots de passe sont chiffrés avec AES-256-GCM à l'aide d'une clé principale stockée dans le trousseau macOS, puis enregistrés dans la zone privée UserDefaults de l'application. La clé principale est générée une seule fois et ne quitte jamais le trousseau. Les mots de passe ne sont jamais stockés en clair ni en Base64 sur le disque. Les mots de passe ne sont pas écrits dans les journaux et ne sont pas envoyés au développeur ou à des tiers.
  • Enregistrements vidéo, clips de lecture téléchargés et instantanés créés par l'utilisateur. Ils sont enregistrés uniquement dans les dossiers ou fichiers que vous choisissez via le sélecteur de fichiers macOS, ou dans un dossier de sauvegarde que vous avez sélectionné et autorisé avec un signet de sécurité.
  • Miniatures locales, état de la fenêtre, préférences d'interface, langue sélectionnée, état du niveau d'achat et autres paramètres de l'application. Ceux-ci restent dans le conteneur d'applications sur votre Mac.

Détection de mouvement et notifications

La détection de mouvement est facultative et s'exécute localement sur votre Mac. L'application analyse les images vidéo sur l'appareil, y compris avec Apple Vision si disponible, pour classer les mouvements tels qu'une personne, un véhicule, un animal ou un mouvement générique. Les cadres ne sont pas téléchargés sur Apple, vers le développeur ou vers un serveur tiers pour cette fonctionnalité.

Si vous activez les alertes de mouvement, l'application peut demander à macOS l'autorisation de notification. Les notifications sont délivrées par macOS et peuvent inclure le nom de l'appareil photo et le sujet détecté. Vous pouvez désactiver les notifications dans System Settings.

Ce que l'application ne fait pas

  • L'application ne collecte pas d'identifiants tels que IDFA, UDID, adresse MAC ou adresse e-mail pour le développeur.
  • L'application n'envoie pas de télémétrie, d'analyses, de statistiques d'utilisation ou de rapports d'erreur au développeur.
  • L'application n'utilise pas de SDK publicitaires, de SDK d'analyse ou de services de profilage.
  • L'application ne télécharge pas les flux de caméras, les instantanés, les enregistrements, les informations d'identification ou la configuration sur un serveur nous appartenant.
  • L'application ne vous suit pas sur les applications ou les sites Web.
  • L'application n'utilise pas les autorisations de la caméra Mac, du microphone, de l'emplacement, des contacts, des calendriers, du Bluetooth ou de la bibliothèque de photos.

Communications réseau

L'application établit uniquement les connexions réseau nécessaires aux fonctionnalités que vous utilisez :

  1. Il se connecte directement aux caméras, NVR et DVR que vous configurez, soit sur votre réseau local, soit via DDNS/redirection de port. Ces connexions peuvent utiliser les flux ONVIF sur HTTP ou HTTPS et RTSP. Le trafic passe directement entre votre Mac et l'appareil que vous avez configuré.
  2. Il utilise des technologies de découverte de réseau local telles que Bonjour et WS-Discovery pour trouver des appareils compatibles sur votre réseau local lorsque vous demandez à l'application d'effectuer une recherche.
  3. Il contacte les services StoreKit de Apple pour les achats intégrés et les vérifications des droits. Le développeur ne reçoit pas votre identifiant Apple, votre numéro de carte ou votre mode de paiement.
  4. Si la synchronisation de l'heure de lecture est activée, l'application peut contacter pool.ntp.org via NTP pour calculer un décalage d'horloge pour une lecture précise de DVR. Cette demande n'est pas utilisée pour le suivi et n'inclut pas les informations d'identification de votre caméra ni la configuration de votre application.
  5. La fonctionnalité Signaler un bug ouvre votre client de messagerie par défaut avec un message pré-rempli uniquement après avoir appuyé sur Envoyer. L'application elle-même ne télécharge pas le rapport.

Sécurité des transports

De nombreuses caméras IP et DVR ne prennent en charge que les certificats HTTP, RTSP ou HTTPS auto-signés. Pour cette raison, l'application autorise les connexions à des appareils photo configurés par l'utilisateur qui peuvent ne pas répondre aux paramètres par défaut d'App Transport Security. Utilisez des réseaux fiables, des mots de passe forts pour les caméras et HTTPS là où votre appareil le prend en charge.

Achats intégrés

Les fonctionnalités Pro et Unlimited sont déverrouillées via le système d'achat intégré de Apple à l'aide de StoreKit. La vérification s'effectue via les API de Apple, y compris Transaction.currentEntitlements et les mises à jour des transactions. L'application stocke localement uniquement l'état des droits résultant afin de pouvoir activer les fonctionnalités achetées.

Autorisations demandées par l'application

  • Local Network : requis par macOS pour découvrir et se connecter aux caméras, NVR et DVR sur votre réseau local. Vous pouvez le révoquer dans System Settings → Privacy & Security → Local Network.
  • Accès aux fichiers sélectionnés par l'utilisateur : utilisé lorsque vous choisissez où enregistrer ou importer des enregistrements, des instantanés ou des fichiers de sauvegarde de l'appareil. Si vous choisissez un dossier de sauvegarde persistant, l'application stocke un signet de sécurité pour ce dossier. L'application n'obtient pas d'accès général au reste de votre disque.
  • Desktop ou autres dossiers : utilisé uniquement lorsque vous choisissez explicitement ce dossier comme destination. L'application ne demande pas d'accès étendu aux dossiers Desktop, Documents ou Downloads.
  • Notifications : facultatives, utilisées pour les alertes de mouvement si vous les activez.
  • Mot de passe Touch ID, Face ID ou Mac : géré par macOS via LocalAuthentication pour les actions sensibles telles que la révélation du mot de passe d'un appareil photo ou l'exportation des données de l'appareil. L'application ne reçoit pas de données biométriques.

Exportation, importation et sauvegarde des configurations

L'application peut exporter la configuration de l'appareil vers un fichier.ipcamvisionprobackup que vous créez et gérez. Les exportations actuelles omettent le champ de mot de passe ONVIF distinct. Toutefois, si vous avez saisi manuellement une URL RTSP, DDNS ou une URL de flux contenant déjà des informations d'identification intégrées, cette URL peut être incluse dans la sauvegarde car elle fait partie de la configuration du périphérique. Si vous importez une ancienne sauvegarde contenant un mot de passe ONVIF, l'application peut la lire une fois pour la migrer vers le stockage d'application privé. Vous êtes responsable de l'endroit où vous stockez les fichiers de sauvegarde.

Signaler une fonctionnalité de bug

La fenêtre Signaler un bug n’envoie pas de données directement depuis l’application. Lorsque vous appuyez sur Envoyer, l'application ouvre votre client de messagerie par défaut avec un e-mail pré-rempli adressé à LorysMay@outlook.it. Vous décidez si vous souhaitez l'envoyer. L'e-mail contient uniquement ce que vous avez tapé, ainsi que la version de l'application et la version macOS pour le débogage.

Enfants

L'application est destinée à un public général. Nous ne collectons pas sciemment de données sur les mineurs car l'application ne collecte pas de données pour le développeur.

Conservation et suppression des données

Les données traitées par l'application restent sur votre Mac jusqu'à ce que vous les supprimiez. Vous pouvez supprimer les données de l'application de Settings → Reset data, supprimer les enregistrements et instantanés générés de leurs dossiers enregistrés ou désinstaller l'application.

Les modifications futures seront publiées dans l'application et, le cas échéant, sur la page canonique de politique de confidentialité en ligne. La date de la dernière mise à jour sera modifiée lorsque la politique sera mise à jour.
