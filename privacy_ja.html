まとめ

IPcamVision Pro は、ユーザー設定の IP カメラ、NVR および DVR を表示、記録、管理するためのローカル macOS アプリです。このアプリは開発者の個人データを収集せず、分析、広告トラッカー、またはプロファイリング サービスを使用せず、カメラ データを当社が所有するサーバーに送信しません。デバイス構成、認証情報、録音、スナップショット、環境設定は、エクスポートまたは共有を選択しない限り、Mac 上に残ります。

アプリがローカルで扱うデータ

アプリは、その機能を提供するために必要なため、デバイス上で次のデータを処理します。

  • カメラ、NVR および DVR のアドレスと構成 (IP アドレスまたはホスト名、RTSP またはストリーム URL、ポート、ストリーム プロファイル、PTZ ブランド、グリッド レイアウト、再生設定、モーション検出設定を含む)。これらの値は、UserDefaults を使用してアプリのプライベート設定に保存され、macOS アプリ サンドボックスによって分離されます。
  • ONVIF ユーザー名とデバイス ユーザー名。これらは、デバイス設定とともにアプリのプライベート設定に保存されます。
  • ONVIF パスワードとデバイス パスワード。パスワードは、macOS キーチェーンに保存されたマスターキーを使用して AES-256-GCM で暗号化され、アプリのプライベート UserDefaults 領域に保存されます。マスターキーは一度だけ生成され、キーチェーンから外に出ることはありません。パスワードは平文または Base64 形式でディスクに保存されることは一切ありません。パスワードはログに書き込まれず、開発者やサードパーティに送信されません。
  • ビデオ録画、ダウンロードされた再生クリップ、およびユーザーが作成したスナップショット。これらは、macOS ファイル ピッカーで選択したフォルダーまたはファイル、またはセキュリティ スコープのブックマークで選択して承認した保存フォルダーにのみ保存されます。
  • ローカルのサムネイル、ウィンドウの状態、インターフェイスの設定、選択した言語、購入層の状態、その他のアプリ設定。これらは Mac 上のアプリ コンテナに残ります。

動きの検出と通知

モーション検出はオプションであり、Mac 上でローカルに実行されます。このアプリは、Apple Vision が利用可能な場合を含め、デバイス上のビデオ フレームを分析して、人、車両、動物、または一般的な動きなどの動きを分類します。この機能のフレームは、Apple、開発者、またはサードパーティのサーバーにはアップロードされません。

モーションアラートを有効にすると、アプリは macOS に通知の許可を求める場合があります。通知は macOS によって配信され、カメラ名と検出された被写体を含めることができます。 System Settings で通知を無効にすることができます。

アプリがやらないこと

  • アプリは、開発者の IDFA、UDID、MAC アドレス、電子メール アドレスなどの識別子を収集しません。
  • アプリは、テレメトリ、分析、使用状況統計、クラッシュ レポートを開発者に送信しません。
  • このアプリは、広告 SDK、分析 SDK、プロファイリング サービスを使用しません。
  • このアプリは、カメラ ストリーム、スナップショット、録画、認証情報、構成を当社が所有するサーバーにアップロードしません。
  • このアプリは、アプリや Web サイト間でユーザーを追跡しません。
  • このアプリは、Mac のカメラ、マイク、位置情報、連絡先、カレンダー、Bluetooth、または写真ライブラリの権限を使用しません。

ネットワーク通信

アプリは、使用する機能に必要なネットワーク接続のみを行います。

  1. ローカル ネットワーク上または DDNS / ポート転送を通じて、設定したカメラ、NVR および DVR に直接接続します。これらの接続では、HTTP または HTTPS および RTSP ストリームを介して ONVIF を使用する場合があります。トラフィックは Mac と設定したデバイスの間で直接送られます。
  2. アプリに検索を依頼すると、Bonjour や WS-Discovery などのローカル ネットワーク検出テクノロジを使用して、LAN 上の互換性のあるデバイスを検索します。
  3. アプリ内購入と権利チェックのために Apple の StoreKit サービスに接続します。開発者は、Apple ID、カード番号、または支払い方法を受け取りません。
  4. 再生時間の同期が有効な場合、アプリは NTP 経由で pool.ntp.org に接続して、正確な DVR 再生のためのクロック オフセットを計算することがあります。このリクエストは追跡には使用されず、カメラの認証情報やアプリ構成は含まれません。
  5. バグを報告する機能では、[送信] を押した後にのみ、デフォルトのメール クライアントが開き、事前に入力されたメッセージが表示されます。アプリ自体はレポートをアップロードしません。

輸送のセキュリティ

多くの IP カメラと DVR は、プレーン HTTP、RTSP、または自己署名 HTTPS 証明書のみをサポートします。そのため、アプリでは、アプリ トランスポート セキュリティのデフォルトを満たしていない可能性があるユーザー構成のカメラ デバイスへの接続が許可されます。デバイスがサポートしている場合は、信頼できるネットワーク、強力なカメラ パスワード、および HTTPS を使用してください。

アプリ内購入

Pro および Unlimited の機能は、StoreKit を使用する Apple のアプリ内購入システムを通じてロック解除されます。検証は、Transaction.currentEntitlements やトランザクション更新などの Apple の API を通じて行われます。アプリは、購入した機能を有効にできるように、結果として得られる権利状態のみをローカルに保存します。

アプリによって要求される権限

  • Local Network: ローカル ネットワーク上のカメラ、NVR および DVR を検出して接続するために、macOS によって必要となります。 System Settings→Privacy & Security→Local Networkで取り消すことができます。
  • ユーザー選択のファイル アクセス: 録画、スナップショット、またはデバイス バックアップ ファイルを保存またはインポートする場所を選択するときに使用されます。永続的な保存フォルダーを選択した場合、アプリはそのフォルダーのセキュリティスコープのブックマークを保存します。アプリはディスクの残りの部分に一般的にアクセスできません。
  • Desktop またはその他のフォルダー: そのフォルダーを宛先として明示的に選択した場合にのみ使用されます。アプリは、広範囲の Desktop、Documents、または Downloads フォルダー アクセスを要求しません。
  • 通知: オプション。有効にした場合、モーション アラートに使用されます。
  • Touch ID、Face ID、または Mac パスワード: カメラのパスワードの公開やデバイス データのエクスポートなどの機密アクションの場合、macOS から LocalAuthentication によって処理されます。アプリは生体認証データを受信しません。

構成のエクスポート、インポート、およびバックアップ

アプリは、ユーザーが作成および管理する.ipcamvisionprobackup ファイルにデバイス構成をエクスポートできます。現在のエクスポートでは、別の ONVIF パスワード フィールドが省略されています。ただし、すでに埋め込み資格情報が含まれている RTSP、DDNS、またはストリーム URL を手動で入力した場合、その URL はデバイス構成の一部であるため、バックアップに含めることができます。 ONVIF パスワードを含む古いバックアップをインポートする場合、アプリはそれを 1 回読み取って、プライベート アプリ ストレージに移行することがあります。バックアップ ファイルを保存する場所については、お客様の責任となります。

バグを報告する機能

[バグを報告] ウィンドウでは、アプリからデータが直接送信されません。 [送信] を押すと、アプリはデフォルトのメール クライアントを開き、LorysMay@outlook.it 宛ての事前入力メールが送信されます。送信するかどうかはあなたが決めます。電子メールには、入力した内容と、デバッグ用のアプリのバージョンと macOS バージョンのみが含まれます。

子供たち

このアプリは一般視聴者を対象としています。アプリは開発者のためにデータを収集しないため、未成年者に関するデータを故意に収集することはありません。

データの保持と削除

アプリによって処理されるデータは、削除するまで Mac に残ります。 Settings → Reset data からアプリ データを削除したり、生成された録画やスナップショットを保存フォルダーから削除したり、アプリをアンインストールしたりできます。
